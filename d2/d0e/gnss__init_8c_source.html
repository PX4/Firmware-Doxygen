121 /****************************************************************************
122  * Name: board_app_initialize
123  *
124  * Description:
125  * Perform application specific initialization. This function is never
126  * called directly from application code, but only indirectly via the
127  * (non-standard) boardctl() interface using the command BOARDIOC_INIT.
128  *
129  * Input Parameters:
130  * arg - The boardctl() argument is passed to the board_app_initialize()
131  * implementation without modification. The argument has no
132  * meaning to NuttX; the meaning of the argument is a contract
133  * between the board-specific initalization logic and the the
134  * matching application logic. The value cold be such things as a
135  * mode enumeration value, a set of DIP switch switch settings, a
136  * pointer to configuration data read from a file or serial FLASH,
137  * or whatever you would like to do with it. Every implementation
138  * should accept zero/NULL as a default configuration.
139  *
140  * Returned Value:
141  * Zero (OK) is returned on success; a negated errno value is returned on
142  * any failure to indicate the nature of the failure.
143  *
144  ****************************************************************************/
145
