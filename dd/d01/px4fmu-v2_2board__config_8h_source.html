94 /* Due to inconsistent use of chip select and dry signal on
95  * different board that use this build. We are defining the GPIO
96  * inclusive of the SPI port and GPIO to help identify pins the
97  * are part of the sensor Net's controlled by different power
98  * domains.
99  *
100  * Only the GPIO_SPIb_xxx_Ppi will be used in the code to insure this are no
101  * cross connections.
102  *
103  * --------------- SPI1 -------------------- SPI4 -------------- Incompatibilities ---------
104  * FMUv2: FmuV3 Cube PixhawkMini
105  * Power Domain: VDD_3V3_SENSORS_EN nVDD_5V_PERIPH_EN V3V:SPI1&SPI4 V3V:SPI1 No SPI4
106  * PA5 SPI_INT_SCK
107  * PA6 SPI_INT_MISO
108  * PA7 SPI_INT_MOSI
109  * PB0 GYRO_DRDY SPI4:EXTERN_DRDY NC
110  * PB1 MAG_DRDY +SPI4:nEXTERN_CS NC
111  * PB4 ACCEL_DRDY NC NC
112  * PC1 Spare ADC ( NC ) +SPI1:SPI_INT_MAG_!CS
113  * PC2 nMPU_CS @MPU6000|MPU9250 @MPU9250
114  * PC13 nGYRO_CS SPI4:nGYRO_EXT_CS NC
115  * PC14 GPIO_EXT_1 nBARO_EXT_CS -20608_DRDY
116  * PC15 nACCEL_MAG_CS SPI4:nACCEL_MAG_EXT_CS 20608_CS
117  * PD7 nBARO_CS
118  * PD15 nMPU_DRDY @MPU6000|MPU9250 @MPU9250
119  * PE2 SPI_EXT_SCK NC
120  * PE4 nSPI_EXT_NSS SPI4:nMPU_EXT_CS NC
121  * PE5 SPI_EXT_MISO NC
122  * PE6 SPI_EXT_MOSI NC
123  *
124  * Notes: Prefixed with @ Does not effect board control
125  * Prefixed with + Input used as Output
126  * Prefixed with - Output used as Input
127  * Prefixed with SPIn: Bus changed
128  *
129  * The board API provides for mechanism to perform a SPI bus reset.
130  * To facilitate a SPI bus reset
131  *
132  * 1) All the pins: SPIn, CD, DRDY associated with the SPI bus are turned to inputs
133  * with outputs driven low. (OFFIng)
134  * 2) The power domain of that bus is turned off.
135  * 3) A usleep it done for ms.
136  * 4) The power domain of that bus is turned back on.
137  * 5) The SPIn pins are re-initialized.
138  * 6) The SPI CS, DRDY pins are re-initialized.
139  *
140  * To insure the complete net is de-energized and is not bing back fed, it is important to
141  * note the all signals in the net list of the parts/bus.
142  *
143  * I.E. Not OFFIng PC1 on V3 would leave that pin back feeding the HMC part. As would not
144  * OFFIng PE4, not associated with SPI1 on V2, but would back feed an MPUxxxx on V3
145  *
146  */
