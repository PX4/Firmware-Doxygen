122  /**
123  * Advertise as the publisher of a topic.
124  *
125  * This performs the initial advertisement of a topic; it creates the topic
126  * node in /obj if required and publishes the initial data.
127  *
128  * Any number of advertisers may publish to a topic; publications are atomic
129  * but co-ordination between publishers is not provided by the ORB.
130  *
131  * The multi can be used to create multiple independent instances of the same topic
132  * (each instance has its own buffer).
133  * This is useful for multiple publishers who publish the same topic. The subscriber
134  * then subscribes to all instances and chooses which source he wants to use.
135  *
136  * @param meta The uORB metadata (usually from the ORB_ID() macro)
137  * for the topic.
138  * @param data A pointer to the initial data to be published.
139  * For topics updated by interrupt handlers, the advertisement
140  * must be performed from non-interrupt context.
141  * @param instance Pointer to an integer which will yield the instance ID (0-based)
142  * of the publication. This is an output parameter and will be set to the newly
143  * created instance, ie. 0 for the first advertiser, 1 for the next and so on.
144  * @param priority The priority of the instance. If a subscriber subscribes multiple
145  * instances, the priority allows the subscriber to prioritize the best
146  * data source as long as its available. The subscriber is responsible to check
147  * and handle different priorities (@see orb_priority()).
148  * @param queue_size Maximum number of buffered elements. If this is 1, no queuing is
149  * used.
150  * @return PX4_ERROR on error, otherwise returns a handle
151  * that can be used to publish to the topic.
152  * If the topic in question is not known (due to an
153  * ORB_DEFINE with no corresponding ORB_DECLARE)
154  * this function will return -1 and set errno to ENOENT.
155  */
156  orb_advert_t orb_advertise_multi(const struct orb_metadata *meta, const void *data, int *instance,
157  int priority, unsigned int queue_size = 1);

182  /**
183  * Subscribe to a topic.
184  *
185  * The returned value is a file descriptor that can be passed to poll()
186  * in order to wait for updates to a topic, as well as topic_read,
187  * orb_check and orb_stat.
188  *
189  * If there were any publications of the topic prior to the subscription,
190  * an orb_check right after orb_subscribe will return true.
191  *
192  * Subscription will succeed even if the topic has not been advertised;
193  * in this case the topic will have a timestamp of zero, it will never
194  * signal a poll() event, checking will always return false and it cannot
195  * be copied. When the topic is subsequently advertised, poll, check,
196  * stat and copy calls will react to the initial publication that is
197  * performed as part of the advertisement.
198  *
199  * Subscription will fail if the topic is not known to the system, i.e.
200  * there is nothing in the system that has declared the topic and thus it
201  * can never be published.
202  *
203  * Internally this will call orb_subscribe_multi with instance 0.
204  *
205  * @param meta The uORB metadata (usually from the ORB_ID() macro)
206  * for the topic.
207  * @return PX4_ERROR on error, otherwise returns a handle
208  * that can be used to read and update the topic.
209  */

217  * orb_check and orb_stat.
218  *

221  *
222  * Subscription will succeed even if the topic has not been advertised;
223  * in this case the topic will have a timestamp of zero, it will never

228  *
229  * Subscription will fail if the topic is not known to the system, i.e.

232  *

328  /**
329  * Set the minimum interval between which updates are seen for a subscription.
330  *
331  * If this interval is set, the subscriber will not see more than one update
332  * within the period.
333  *
334  * Specifically, the first time an update is reported to the subscriber a timer
335  * is started. The update will continue to be reported via poll and orb_check, but
336  * once fetched via orb_copy another update will not be reported until the timer
337  * expires.
338  *
339  * This feature can be used to pace a subscriber that is watching a topic that
340  * would otherwise update too quickly.
341  *
342  * @param handle A handle returned from orb_subscribe.
343  * @param interval An interval period in milliseconds.
344  * @return OK on success, PX4_ERROR otherwise with ERRNO set accordingly.
345  */
