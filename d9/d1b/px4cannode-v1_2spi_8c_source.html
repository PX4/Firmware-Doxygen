95 /****************************************************************************
96  * Name: stm32_spi1/2/3select and stm32_spi1/2/3status
97  *
98  * Description:
99  * The external functions, stm32_spi1/2/3select and stm32_spi1/2/3status
100  * must be provided by board-specific logic. They are implementations of
101  * the select and status methods of the SPI interface defined by struct
102  * spi_ops_s (see include/nuttx/spi/spi.h). All other methods (including
103  * stm32_spibus_initialize()) are provided by common STM32 logic. To use
104  * this common SPI logic on your board:
105  *
106  * 1. Provide logic in stm32_boardinitialize() to configure SPI chip
107  * select pins.
108  * 2. Provide stm32_spi1/2/3select() and stm32_spi1/2/3status() functions
109  * in your board-specific logic. These functions will perform chip
110  * selection and status operations using GPIOs in the way your board
111  * is configured.
112  * 3. Add a calls to stm32_spibus_initialize() in your low level application
113  * initialization logic
114  * 4. The handle returned by stm32_spibus_initialize() may then be used to
115  * bind the SPI driver to higher level logic (e.g., calling
116  * mmcsd_spislotinitialize(), for example, will bind the SPI driver to
117  * the SPI MMC/SD driver).
118  *
119  ****************************************************************************/
