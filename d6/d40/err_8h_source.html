34 /**
35  * @file err.h
36  *
37  * Simple error/warning functions, heavily inspired by the BSD functions of
38  * the same names.
39  *
40  * The err() and warn() family of functions display a formatted error
41  * message on the standard error output. In all cases, the last
42  * component of the program name, a colon character, and a space are
43  * output. If the fmt argument is not NULL, the printf(3)-like formatted
44  * error message is output. The output is terminated by a newline
45  * character.
46  *
47  * The err(), errc(), verr(), verrc(), warn(), warnc(), vwarn(), and
48  * vwarnc() functions append an error message obtained from strerror(3)
49  * based on a supplied error code value or the global variable errno,
50  * preceded by another colon and space unless the fmt argument is NULL.
51  *
52  * In the case of the errc(), verrc(), warnc(), and vwarnc() functions,
53  * the code argument is used to look up the error message.
54  *
55  * The err(), verr(), warn(), and vwarn() functions use the global
56  * variable errno to look up the error message.
57  *
58  * The errx() and warnx() functions do not append an error message.
59  *
60  * The err(), verr(), errc(), verrc(), errx(), and verrx() functions do
61  * not return, but exit with the value of the argument eval.
62  *
63  */
